1. Introduction

The objective of this project was to Build and Deploy a Machine Learning Model for Predicting House Prices. The project utilizes the California Housing Dataset from Scikit-learn, which contains information about housing characteristics and median house values for different districts in California. This dataset was derived from the 1990 U.S. Census and includes important demographic and housing features.

The project follows a structured approach, including data preprocessing, feature engineering, model selection and optimization, deployment of a Flask-based API on AWS, and frontend integration to create a user-friendly interface. The final system provides real-time predictions for house prices based on user inputs.

2. Data Preprocessing and Feature Engineering

The dataset consists of 8 predictive attributes and a target variable that represents the median house value for California districts.

Dataset Overview

Number of Instances: 20,640
Number of Attributes: 8 numeric predictive attributes and 1 target variable
Attributes:
- MedInc: Median income in block group
- HouseAge: Median house age in block group
- AveRooms: Average number of rooms per household
- AveBedrms: Average number of bedrooms per household
- Population: Block group population
- AveOccup: Average number of household members
- Latitude: Block group latitude
- Longitude: Block group longitude

Handling Missing Values

The dataset was checked for missing values, and none were found. Therefore, no imputation was necessary.

Feature Engineering

To enhance the predictive power of the model, additional features were created:

  • RoomsPerBedroom: Ratio of average rooms to average bedrooms.
  • PopulationPerBedroom: Ratio of population to average bedrooms.
  • MedInc_HouseAge: Interaction feature combining median income and house age.
  • Latitude_Longitude: Product of latitude and longitude to capture geographic effects.

Feature Scaling

Standardization was applied to numerical attributes to ensure uniform scaling, improving model performance.

3. Model Selection and Optimization

Model Comparison

Several machine learning models were evaluated based on their performance metrics:

Model Mean Absolute Error (MAE) Root Mean Squared Error (RMSE) R² Score
XGBoost 0.2874 0.4406 0.8518
Random Forest 0.3273 0.5036 0.8064
Decision Tree 0.3956 0.5954 0.7295
Linear Regression 0.5227 0.7233 0.6007

Final Model Selection

Based on the evaluation metrics, the XGBoost model was selected as the best-performing model, achieving the highest R² score (0.8518) and the lowest MAE and RMSE values.

Selected Features

The XGBoost model used all available features for training, ensuring maximum predictive power.

Several machine learning models were evaluated to determine the best-performing one for predicting house prices. The models considered included:

  • Linear Regression: A baseline model to establish a simple relationship between features and house prices.
  • Decision Tree Regressor: A non-linear model that captures complex feature interactions.
  • Random Forest Regressor: An ensemble approach that reduces overfitting by averaging multiple decision trees.
  • XGBoost Regressor: A gradient boosting model known for high performance and optimization.

Hyperparameter Optimization

To improve model performance, hyperparameters were fine-tuned using:

  • Grid Search: Exhaustive search over a predefined set of hyperparameter values.
  • Randomized Search: Random sampling of hyperparameters to find optimal configurations faster.

Model Evaluation

The models were assessed using the following metrics:

  • Mean Squared Error (MSE): Measures the average squared difference between predicted and actual values.
  • Mean Absolute Error (MAE): Measures prediction errors in absolute terms.
  • R-squared Score (R²): Determines how well the model explains variance in house prices.

Model Storage and Tracking

The trained models were stored using joblib for future use, and MLflow was used for tracking experiments and comparing different model versions.

5. Frontend Interface

The frontend is a lightweight and user-friendly web interface designed to allow users to input housing-related data and receive predictions from the trained model.

Features and Design

  • HTML Form for Input: The interface includes an input form with fields for key housing attributes such as median income, house age, average rooms, and geographical coordinates.
  • CSS Styling: Basic styling ensures a clean and user-friendly layout with a centered form and responsive design.
  • JavaScript for API Interaction:
    • Captures user input from the form.
    • Sends a POST request to the API endpoint.
    • Handles API responses and displays the predicted price dynamically.
    • Displays error messages if the API call fails.

API Integration

The frontend communicates with the `/predict` endpoint by sending user-provided inputs in JSON format. The API then returns the predicted house price, which is displayed on the webpage.

Updated API URL:

6. Conclusion

The California Housing Price Prediction System successfully integrates data preprocessing, machine learning model selection, deployment through Flask and AWS, and a user-friendly frontend. The deployment strategy ensures that the API is accessible via a cloud-hosted endpoint, allowing seamless interaction with the frontend interface. Users can input housing attributes, send requests to the API, and receive real-time predictions.
